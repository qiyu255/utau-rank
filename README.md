# 视频帧自动筛选与分组流水线

## 1. 项目概述

本项目实现了一套**自动化图像处理流水线**，用于从一组视频帧（或连续拍摄的图像序列）中筛选出**质量最佳、内容一致且具有代表性**的帧，并根据**前景颜色**进行分组，最终输出每组的“最佳代表帧”。

典型应用场景：**文档扫描、票据识别、屏幕录制关键帧提取**等，要求从大量相似图像中挑选出清晰、无遮挡、无重复的高质量样本。

---

## 2. 整体流程

流水线由 6 个顺序执行的模块组成（定义在 `run.py` 中）：

```
原始帧 → ch00_ready → ch01_clean_daekness → ch02_clean_miss → ch03_clean_same → ch04_group → (ch90_get_best)
```

- **ch00_ready**：裁剪并拼接 ROI（感兴趣区域），生成标准化图像。
- **ch01_clean_daekness**：基于亮度/暗度特征，剔除过暗（欠曝）的帧。
- **ch02_clean_miss**：基于文本行区域特征，剔除内容缺失或排版异常的帧。
- **ch03_clean_same**：检测并去除重复/高度相似的帧，保留每个相似簇中“最稳定”的代表帧。
- **ch04_group**：根据前景颜色（如背景色）对帧进行聚类分组。

## 3. 模块详细说明

### 3.1 公共模块 `common.py`

提供全局工具函数、日志记录器和数据存储类。

| 功能 | 说明 |
|------|------|
| `get_boxes(name, width, height)` | 从 `data/roi_*.json` 加载 ROI 配置，并按当前图像尺寸自动缩放。支持 `'layout'`（布局区域）和 `'line'`（文本行区域）两种预设。 |
| `crop(img, box)` | 安全裁剪图像，自动边界检查与钳制。 |
| `concat(images, bg)` | 将多张图像水平拼接，背景色可自定义。 |
| `Store` 类 | 将统计数据和分组结果保存为 CSV / JSON，便于前端可视化或二次分析。 |
| `setup_logger()` | 配置控制台（INFO 级）和文件（DEBUG 级）双路日志。 |

**数据目录**：
- `cache/` – 中间缓存（CSV、JSON、pickle）
- `data/` – 存放 ROI 配置文件（需用户提供）
- `logs/` – 运行日志

---

### 3.2 ch00_ready – 图像标准化

**目的**：将原始帧裁剪出“标题”和“统计区域”，旋转并水平拼接，统一后续分析的目标区域。

**处理逻辑**：
1. 加载布局 ROI（`layout`），获取 `'title'` 和 `'stats'` 区域。
2. 分别裁剪并旋转标题区（顺时针 90°）。
3. 将标题与统计区水平拼接（背景白色），输出标准化图像。

**输入/输出**：输入原始帧路径列表，输出 `cache/ready/` 下同名的处理后的图像。

---

### 3.3 ch01_clean_daekness – 暗帧过滤

**目的**：剔除曝光不足、整体过暗的帧。

**特征提取**（共 9 维）：
- 灰度均值、暗像素比例（≤50）、直方图质心
- HSV 的 V 通道均值、LAB 的 L 通道均值
- 极暗像素比例（≤30）、暗区聚集度、最大暗区面积比

**评分机制**：加权融合（权重预设），经 Sigmoid 映射为 0~1 的成本值，值越低表示帧越亮。通过阈值 `MASK_BY = 0.03445` 筛选出合格帧。

**输出**：筛选后的“明亮”帧列表，并保存统计 CSV 及好坏列表 JSON。

---

### 3.4 ch02_clean_miss – 内容缺失过滤

**目的**：剔除因遮挡、裁切或排版错位导致内容不完整的帧。

**特征提取**（基于 ROI `'line'` 区域）：
- 对每个候选行区域，计算其垂直投影的行前景比例。
- 提取以下 6 个特征：
  - 浓度（主波段集中度）
  - 水平连续性（行内最长连续前景）
  - 峰值（最大前景比例）
  - 背景纯度（主波段外最大前景比例）
  - 离散度（前景散布程度）
  - 波段比（主波段高度占比）

**评分**：组合为成本函数，成本越低表示排版越规整。通过阈值 `MASK_BY = 0.25` 筛除不良帧。

**输出**：筛选后的帧列表及统计结果。

---

### 3.5 ch03_clean_same – 相似帧去重

**目的**：将高度相似的帧（如静止画面）归为一组，并为每组选取“最稳定”的代表帧，消除冗余。

**相似度度量**：
- 结构相似性（SSIM）及其分量（亮度、对比度、结构）
- 像素级差异（PSIM）：高斯模糊后二值化差异图的比例和均值

**聚类策略**：
- 计算相邻帧的综合相似度 `sim = 0.1*PSIM + 0.9*SSIM`。
- 设定阈值 `SAME_THRESH = 0.93`，将相似度 ≥ 阈值的帧视为同一组（相似组）。
- 对每个相似组，利用滑动窗口方差寻找最平坦（稳定）的区间，选取该区间最后一帧作为代表。

**输出**：每组一个代表帧的列表，并保存分组信息（JSON）便于可视化。

---

### 3.6 ch04_group – 按颜色聚类

**目的**：根据图像前景（内容区）的颜色特征，将帧分成若干颜色组（例如不同背景色或不同文档类型）。

**特征**：
- 对 ROI `'line'` 区域提取前景像素（通过 HSV 饱和度/亮度阈值）
- 在 LAB 空间计算前景的 **a* 和 b* 通道均值和标准差**（4 维特征）。

**聚类**：
- PCA 降维（最多 4 维）
- KMeans 聚类（默认 5 类，实际类数 ≤ 图像数）
- 每组存储其平均颜色（BGR → HSV）。

**输出**：分组后的帧列表（每组一个列表），以及每组平均 HSV 颜色值，保存为 JSON。

## 5. 使用指南

### 5.1 准备工作
1. 将所有待处理的原始帧放入 `cache/frames/` 目录（支持 PNG、JPG 等）。
2. 在 `data/` 目录下准备 ROI 配置文件：
   - `roi_layout.json`：定义 `'title'` 和 `'stats'` 区域（用于 ch00）。
   - `roi_line.json`：定义文本行区域（用于 ch02 和 ch04）。
   - 配置文件格式参考：
```json
{
  "referenceWidth": 1920,
  "referenceHeight": 1080,
  "boxes": [
    {"name": "title", "x": 100, "y": 50, "w": 200, "h": 50},
    ...
  ]
}
```



### 5.4 查看结果
- **中间统计**：`cache/csv/` 下各模块的 CSV 文件，可查看特征分布。
- **分组结果**：`cache/result/` 下的 JSON 文件，包含每组的帧列表和代表帧。
- **日志**：`logs/latest.log` 记录详细运行信息。

| 文件 | 内容 |
|------|------|
| `cache/ready/*.png` | 标准化后的图像（ch00 输出） |
| `cache/result/ch01_clean_daekness_*.json` | 好/坏帧列表 |
| `cache/result/ch02_clean_miss_*.json` | 好/坏帧列表 |
| `cache/result/ch03_clean_same.json` | 去重后的分组及代表帧 |
| `cache/result/ch04_group.json` | 颜色分组及每组平均色 |
| `cache/csv/*.csv` | 各阶段特征统计表（可用于分析阈值） |

---

## 7. 自定义与调参

- **阈值调整**：各模块顶部定义的常量（如 `MASK_BY`, `SAME_THRESH`, `N_CLUSTERS` 等）均可按需修改。
- **特征权重**：`ch01` 和 `ch02` 中的 `WEIGHTS` 数组可调整以改变筛选倾向。
- **ROI 区域**：通过修改 JSON 文件可适配不同布局的图像。
- **启动阶段**：修改 `run.py` 的 `start` 变量，配合缓存可加速调试。

## 十、落地实施步骤建议

1. **第1天**：选取10个典型视频，手工走通系列A至E，验证PaddleOCR在特殊数字上的初始效果，确定是否需要微调。
2. **第2-3天**：开发氛围色提取、封面图剔除、模板聚类代码，完成首批模板的ROI标注。
3. **第4-5天**：完善图像增强管线，若数字识别率不足80%，采集数字样本微调模型。
4. **第6天**：搭建分布式处理脚本，小批量试跑500个视频，优化性能瓶颈。
5. **第7天起**：全量运行，人工审核队列跟进，必要时迭代模型。

这套方案将你提供的信息——氛围色、封面图、数字特殊字体描边、趋势箭头、主题结构——全部转化为可编程的信号，最大程度减少人工，确保3万个视频的高效准确提取。如你需要某一环节的代码示例或进一步调优，可随时继续讨论。
