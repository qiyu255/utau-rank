

## 一、整体流水线概览

```text
系列A：视频预处理与帧抽取
系列B：页面类型判别与主题切分（融合氛围色）
系列C：模板发现、聚类与ROI定义（排除封面图）
系列D：图像增强专用管线（去背景、去描边、超分、去爆浆）
系列E：多帧OCR融合与趋势符号识别
系列F：结构化输出与质量控制
系列G：工程化并行与人工闭环
```

每个系列都输出标准化的中间产物，下游统一消费。

## 二、系列A：视频预处理与最佳帧抽取

**目标**：将3万个视频高效转化为每个数据页的“清晰代表帧”，同时保留页面帧簇用于后续多帧融合。

**工具**：`ffmpeg`、Python（OpenCV、imagehash）

**处理步骤**：
1. **关键帧抽取**  
   - 使用 `ffmpeg` 按固定时间间隔（如0.5秒）提取帧，避免全解码：  
     `ffmpeg -i input.mp4 -vf fps=2 frame_%04d.png`  
   - 若已知视频帧率，可直接用 `select` 过滤器减少重复帧。

2. **镜头/页面分割**  
   - 计算相邻帧的感知哈希（pHash）差异，差异低于阈值（5%）的归为同一页面簇。  
   - 同时可利用**氛围色变化**辅助：若相邻帧的主色调直方图发生剧烈跳变，且随后稳定，强制切开为新页面（应对淡入淡出导致的哈希渐变）。

3. **代表帧选择**  
   - 对每个页面簇，计算每帧的拉普拉斯方差（模糊度指标），取最大值作为代表帧。  
   - 若整簇最大方差仍低于全局阈值（爆浆页），则标记该簇为“低质量簇”，输出所有帧到系列D进行多帧合成，不单靠代表帧。

**输出**：每个视频的页面簇信息文件（JSON），包含帧路径、时间戳、清晰度得分、主色调向量。

## 三、系列B：页面类型判别与主题自动切分

**目标**：将页面分为「标题页」「数据页」「其它」，并利用氛围色+关键词自动切分出不同主题段。

**工具**：PaddleOCR（文本检测）、OpenCV（颜色统计）、轻量级分类器（MobileNetV3 可选）

**详细流程**：
1. **标题页快速过滤**  
   - 对代表帧运行 PaddleOCR 文本检测（不识别），若检测到的文本行中，经简单关键词匹配命中 `["第", "主题", "回顾", "总览", "目录"]` 且无表格式文本块（通过文本框宽高比、排列判据），则判为标题页，直接丢弃。  
   - 为提高鲁棒性，可训练一个二分类图像分类器，输入为代表帧缩略图，输出标题页/数据页。训练样本可从首批视频中手工标注几百张获得。

2. **氛围色提取与主题边界检测**  
   - **主色调提取**：将代表帧转换到 HSV，排除过暗/过亮像素，对 H 和 S 通道做二维直方图统计，取峰值作为该页的“氛围色向量”。  
   - **边界判定**：沿时间线计算相邻页氛围色向量的余弦距离，若距离突增且后续页维持新色调，即标记为主题切换点。  
   - **双验证**：切换点若同时检测到标题页，置信度极高；若仅有色调跳变，也视为潜在边界，交由系列C的模板匹配二次确认。

3. **主题标注**  
   - 运行初期人工建立颜色-主题映射表：  
     ```
     色相150°浅蓝 → 主题“季度销售”  
     色相45°浅橙 → 主题“市场趋势”  
     色相0°浅红 → “去年回顾”  
     ```  
   - 后续所有视频自动套用，未知氛围色标记为“特殊主题X”，暂存待人工定义。

**输出**：带主题标签的页面序列，每个页面有代表性帧、氛围色、所属主题ID。

## 四、系列C：数据页模板发现、ROI定义与封面图排除

**目标**：同一主题内，利用氛围色和布局特征聚类出固定模板，自动避开封面图片，定义精确的数据提取区域（ROI）。

**工具**：PaddleOCR（版面分析）、PP-Structure、OpenCV、Labelme（标注）、scikit-learn（聚类）

**详细流程**：
1. **封面图片自动剔除（先决步骤）**  
   在模板聚类前，先对每个数据页代表帧进行**版面分析**：  
   - 用 PaddleOCR 的版面分析模型（或 PP-Structure 的版面恢复）输出“表格”“文字”“图片”区域。  
   - 将标签为“图片”且面积大于画面 15% 的区域用白色填充，得到无封面图的干净帧。  
   - 若封面图上有叠加文字被误留，可结合**文本框密度**进一步滤除：密度极低的边缘区域视为残余图片剔除。  
   这个步骤保证后续布局特征提取不受封面图干扰。

2. **布局特征提取**  
   - 在清除封面图后的图像上，再次运行 PaddleOCR 文本检测，提取所有文本框的归一化中心坐标 (x,y,w,h)。  
   - 对于表格区域，也可用 PP-Structure 的表格识别结果，抽取单元格坐标作为特征。  
   - 布局向量由这些坐标的分布统计量构成（如分块网格占有率、行列直方图）。

3. **主题内联合聚类（颜色 + 布局）**  
   - 特征向量 = [氛围色H, 氛围色S, 布局坐标分布的前20个PCA分量]。  
   - 使用 DBSCAN（或层次聚类）对同一主题下的所有页面进行聚类，eps 值适当，每个簇对应一个数据页模板。  
   - 跨视频复用：不同视频但氛围色相同且聚为一类的页面，强制归属同一模板。

4. **ROI人工定义（一次性工作）**  
   - 对每个模板簇，抽取一张最清晰帧（已去封面图），使用 Labelme 或 LabelImg 框选所有数据字段区域：  
     - 文本标题、数字字段（可能描边）、趋势箭头区域、“NEW”标识区域。  
   - 同时生成一个二值掩膜，保留这些ROI区域，其余全部置黑/白。  
   - 保存为模板配置文件（JSON）：模板ID、所属主题、氛围色范围、ROI坐标与类型、掩膜路径。

**输出**：模板配置库，每个模板绑定了ROI定义和封面图排除方案。一个新页面只需匹配模板ID即可继承这些定义。

## 五、系列D：图像增强专用管线

**目标**：针对特殊数字、描边、小字体、爆浆画质，在OCR前极大提升ROI区域的可读性。

**工具**：OpenCV、Real-ESRGAN（超分）、SwinIR、光流对齐（cv2.optflow）

**按页面质量分两条线**：

### 线路1：常规页面（清晰度达标）
1. **去背景与强化对比**  
   - 已知该主题氛围色（淡色背景），在HSV空间设阈值将该背景色像素直接置为白色。  
   - 同时可根据表格线颜色（通常与氛围色同色系更深）将其淡化，避免OCR误识。  
2. **去描边预处理**  
   - 对数字ROI区域，若描边是固定颜色（如白底红描边），可做颜色筛选；若为轮廓描边，用形态学“顶帽”运算提取描边层，从原图中减去。
3. **超分辨率增强**  
   - 对小文本ROI（高度<20px）使用 Real-ESRGAN 的通用模型做2倍放大，恢复笔画。
4. **二值化**  
   - 使用自适应高斯阈值（或 Otsu）将文字转为纯黑，背景纯白。

### 线路2：爆浆帧簇（低质量簇）
1. **多帧对齐与合成**  
   - 使用光流（DualTVL1）将该簇内所有帧对齐到代表帧。  
   - 像素级取中值或平均值，抹去随机压缩噪点和局部马赛克，得到一张干净的合成图。
2. **合成后再送入线路1的处理流程**。

**输出**：每个ROI区域的增强后切片图，直接喂给OCR。

## 六、系列E：多帧OCR融合与趋势符号识别

**目标**：利用页面停留多帧的优势，用投票机制得到高置信度文本，同时独立识别趋势箭头和NEW标识。

**工具**：PaddleOCR（文本识别）、自定义CRNN（微调可选）、模板匹配（箭头）

**流程**：
1. **多帧OCR**  
   - 对同一页面的所有帧（或采样3~5帧），分别应用系列D的增强管道，得到每个ROI的切片。  
   - 对每个切片用 PaddleOCR 识别，限制字符集：数字字段设 `[0-9.,%百千万亿]`，文本字段不限。  
   - 收集多帧结果，取置信度最高或多数投票的字符串。如果某帧结果为空但其它帧有，丢弃空值。
2. **数字专用修正**  
   - 若通用OCR对特殊数字仍不佳，收集该模板的数字切片（几百张），微调一个 CRNN 模型（PaddleOCR 支持微调）。微调后替换该模板的数字识别器。
3. **趋势符号识别**  
   - **箭头**：提前保存向上/向下箭头模板（从清晰帧截取），对趋势ROI使用多尺度模板匹配，分三类：向上、向下、无。  
   - **“NEW”标识**：同样用 PaddleOCR 识别，同时辅以颜色检测（如红色区域+文字“NEW”），双重确认。
4. **数据校验**  
   - 对数字字段检查值域（如百分比0-100），异常值标记待复核。

**输出**：每个数据页的结构化JSON，包含字段名、数值、趋势、置信度。

## 七、系列F：结构化输出与质量控制

**目标**：将海量JSON汇总为可用的数据库，并自动标记低质量结果送人工。

**工具**：Python脚本、SQLite/PostgreSQL、Streamlit（人工审核界面）

**流程**：
1. **结构化存储**  
   - 每条记录包含：视频来源、时间戳、主题、模板ID、各字段值、趋势、OCR置信度、代表帧路径。
2. **自动质量标记**  
   - 任一字段置信度<0.7 或 多帧结果冲突 或 数字超出合理范围，标记为“待审核”。  
   - 所有特殊主题（未映射氛围色）也进入审核列。
3. **人工审核界面**  
   - 使用 Streamlit 快速搭建，显示代表帧图片（覆盖ROI框）与提取结果，人工一键确认或修改。  
   - 修正后的数据可作为微调模型的训练样本，形成闭环迭代。

## 八、系列G：工程化并行与任务调度

**目标**：3万个视频在可接受的时间内处理完毕。

**工具**：Apache Airflow 或 Celery + Redis（任务分发）、Docker（环境一致）、GPU集群

**架构**：
- 每个视频作为一个任务，分发到GPU节点。  
- 节点内流水线：系列A->B->C->D->E 串行，但多视频并行。  
- 所有模型（OCR、超分）支持GPU批量推理，PaddleOCR 可以一次处理多张切片。
- 中间结果按视频哈希缓存，失败重试只跑未完成部分。
- 预估：单张V100显卡，处理一个5分钟视频约需2-3分钟（含超分），3万视频以100张卡并行，约需10小时；若只10张卡，约4天完成，完全可接受。

## 十、落地实施步骤建议

1. **第1天**：选取10个典型视频，手工走通系列A至E，验证PaddleOCR在特殊数字上的初始效果，确定是否需要微调。
2. **第2-3天**：开发氛围色提取、封面图剔除、模板聚类代码，完成首批模板的ROI标注。
3. **第4-5天**：完善图像增强管线，若数字识别率不足80%，采集数字样本微调模型。
4. **第6天**：搭建分布式处理脚本，小批量试跑500个视频，优化性能瓶颈。
5. **第7天起**：全量运行，人工审核队列跟进，必要时迭代模型。

这套方案将你提供的信息——氛围色、封面图、数字特殊字体描边、趋势箭头、主题结构——全部转化为可编程的信号，最大程度减少人工，确保3万个视频的高效准确提取。如你需要某一环节的代码示例或进一步调优，可随时继续讨论。
